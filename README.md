# 日本麻將 AI 開發與多策略對抗分析

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
動機: 日本麻將具備比台灣麻將還要複雜的胡牌(獲勝)規則，且在看不到其他人手牌的情況之下充滿不確定性，這使得策略在打日本麻將時顯得重要。我自己會玩日本麻將，也希望能找到比較適合的遊玩策略。我想在專案中實作具有不同策略性質的AI，實際讓他們各自對戰並收集數據，驗證可透過更有邏輯性的策略來平衡運氣，並嘗試比較不同的策略的優劣性。

目標:
1. 建構高效能的決策引擎: 利用 C++ 的運算優勢，開發拆解手牌及計算最佳策略的演算法，讓AI在對局中處理複雜的牌局狀態並做出邏輯回饋。
2. 多種測略模擬: 實作三種不同策略邏輯的模型。
3. 量化策略對隨機性的影響: 藉由大規模的自動化對局實驗，收集具統計意義的數據（如勝率、放槍率、平均順位）。透過數據分析，量化不同邏輯策略的實質表現。

### 競品比較

Suphx、Mortal等麻將AI引擎: 根據許多牌譜，採用強化學習實作的大模型，決策邏輯是根據AI的機率模型來決定。這類型的AI勝率毫無疑問會超過我使用較簡單的邏輯和啟發式實作出的AI，但我在這個專案的目標是"自己寫出拆解手牌的邏輯"(如向聽數計算、有效張數計算、權重設定等)，以及"尋找較大方向能穩定加分策略"給我自己遊玩時參考，為此，採用權重設定的方式較能模擬人腦在打牌時的思考方式，而非做出一個追求極致勝率的黑盒子模型(涉及太過複雜、人腦難以模擬的機率計算)。

### 預期功能
<!-- 列出預計實作的功能 -->
1. 實作基礎遊戲引擎，支援日本麻將的操作(打牌、吃、碰、槓等)與核心規則(胡牌判斷、分數判斷等)，讓AI之間能夠互相對戰，若時間允許，可新增圖形介面，並支援單人與電腦來對戰。
2. 實作三種不同邏輯的AI:
   
   (1) 隨機: 每一輪隨機打出一張手上的牌，沒有任何策略，作為對照組用。
   
   (2) 初階邏輯: 僅計算向聽數(距離聽牌還差幾張)，每一輪追求向聽數最小化。
   
   (3) 進階邏輯: 除了基礎的向聽數外，增加有效張數(某種牌最多剩餘幾張)、權重評分、基礎防守。
   
3. 實驗數據產生及分析:

   (1) 驗證擁有基礎邏輯會比隨機亂打好，以平均順位(拿到第幾名)來比較。

   (2) 驗證在有考慮進階數據的情況下會比只有基本邏輯好，預期進階邏輯的得分會更高、放槍率更低、且四位率(最後一名)會較低。

   (3) 調整進階邏輯權重以模擬不同策略(如追求高分、快速胡牌、防守等)，比較各種策略的優劣性，首先觀察各策略的代表數據是否更佳(如高分就是看平均點數)，最後依據市面上遊戲加扣分的機制(詳見註1)算出各場平均，進行各策略綜合評價。

註1: 

$$
PT_i = \text{順位加成} + \frac{\text{終局點數} - 25,000}{1,000}
$$

$$
\overline{PT} = \frac{1}{N} \sum_{i=1}^{N} PT_i
$$

*本公式是參考市面麻將遊戲"雀魂麻將"段位的加扣分公式。*

註2: 考量到專案時間限制，在實作中會略為簡化規則與邏輯，規則的部分會捨棄某些現實機率太低(數據上低於萬分之一)的役種(胡牌的牌型)判斷，而邏輯上也會讓AI傾向去做比較常見的役種。

### 實驗設計

我預計將實驗分為三階段，來嘗試驗證邏輯性可以對抗無法掌控的運氣，以及比較不同策略。

1. 基礎邏輯驗證: 在對戰中放入兩隻隨機AI和兩隻基礎邏輯AI，互相對戰5000場，將對戰結果及統計數據(勝率、放槍率、平均順位)寫進檔案進行分析，目標為驗證有基本邏輯會比完全隨機來的好。
2. 進階邏輯驗證: 在對戰中放入一隻隨機AI、一隻基礎邏輯AI、和兩隻進階邏輯AI，互相對戰5000場，將對戰結果及統計數據(勝率、放槍率、平均順位、平均聽牌巡數、獲得分數)寫進檔案進行分析，目標為驗證進階的邏輯可以更有效率的胡牌及獲得更好的分數。
3. 策略比較: 在對戰中全部放入進階邏輯AI，但各自調整不同的權重以模擬不同的策略(如較偏重得高分、較偏重快速聽牌或較偏重防守等)，互相對戰5000場，將對戰結果及統計數據(勝率、放槍率、平均順位、平均聽牌巡數、獲得分數)寫進檔案進行分析，目標為比較不同策略能產生的效果。

註: 考慮到此部分(實驗設計)寫在其他段落會使其有點過於冗長，因此在這邊新增原始模板沒有的段落。

### 目前進度
<!-- 完成了什麼 -->
1. 基本資料結構 - 牌面、手牌、副露牌(透過吃、碰、槓會讓其他玩家看到的牌)。
2. 初始遊戲引擎實作 - 可以洗牌、發牌、引入四隻AI進行對局，並判定有無玩家胡牌。

3. 向聽數邏輯計算(向聽數的意思是"最少差幾張就可以聽牌")。

   (1) 特殊牌型直接用枚舉計算

   (2) 一般牌型利用DFS遞迴找出向聽數
   
4. 實作隨機AI和基礎邏輯AI，利用polymorphism繼承自Player類別，其中基礎邏輯AI會盡可能最小化向聽數
   
5. 簡單驗證牌局可正常進行，且基礎邏輯AI確實能照著目標邏輯打牌(註: 目前驗證不是很嚴謹，只有手動查看，之後將會做更嚴謹的驗證)。


### 遇到的困難
<!-- 遇到什麼問題、如何解決或打算如何解決 -->
1. 問題: 我剛開始實做的時候覺得麻將規則有點複雜，吃碰槓等操作和有效張數、役種判定及分數判定的邏輯比較難實作。
   
   解決: 我暫時把這部分延後實作(將留到下一階段再進行實作)，因為這階段我是要先做出隨機AI和基礎邏輯AI，其中基礎邏輯僅是最小化向聽數，還沒有涉及到上述比較複雜的實作，因此我把這階段的目標先著重在可以正確執行對局並寫好向聽數的邏輯。

2. 問題: 我第一次實作向聽數判定邏輯時會出現向聽數不在該有的範圍之內(例如-2, -3等，"該有的範圍"是\[-1, 8]，其中-1代表已經胡牌)。
   
   解決: 後來發現在計算時少判定"面子"(ABC三張連續或AAA三張相同)+"搭子"(AB兩張連續、AC崁張、AA兩張相同)應小於等於4，修正邏輯後就能跑出我預期的結果。

3. 問題: 我做出的向聽數判定邏輯難以驗證他的正確性(不知道輸出的結果是不是真正的向聽數)。

   解決: 我一開始嘗試請LLM模型幫忙生成一些測資，但效果不佳(有答案給錯或牌根本對不上的情況)，到最後那些測資還是只能手動查看。後來發現網路上有現成的驗證器(由麻將遊戲的官方製作)，改用驗證器來和我的程式執行結果對比。

## Final Report

### 專案說明
<!-- 完整描述你的專案做了什麼 -->

本專案實作一種基礎邏輯AI和三種進階邏輯AI，並提供引擎讓AI對戰並記錄數據，以下針對AI實作與實驗數據進行詳細說明。

AI實作 - 
1. 向聽數AI

   枚舉每一張牌計算向聽數(邏輯可參考prototype report或程式碼)，選取最小的打出。

   優點: 速度快，較能穩定的往胡牌發展。

   缺點: 邏輯太過單一且沒有考慮有效張數，如果後期牌都差不多打光可能會聽一張根本已經沒有的牌。

2. 速度AI

   (1) 以最小化向聽數為主，但多考慮有效張數(畢竟有的牌可能被打光了，就該視為不存在這張牌)。
   
   (2) 在不影響有役的情況下嘗試吃碰槓。
   
   (3) 考慮和某些好做役種的距離，傾向去做那些役
   
   (4) 權重公式 : `(8 - 向聽數)x1000 + 役種距離加成(約100~500之間) + 寶牌加成(100)  + 有效張數x10`

   枚舉每張牌模擬打出，計算權重(越高代表打掉這張越好)，最後打出權重最高者。

   優點: 能比向聽數AI更有效率的追求速度。

   缺點: 在跟別人速度差不多的情況下放槍率可能較高。

3. 防守AI

   (1) 在別人可能或確定聽牌(副露很多或宣告立直)之後考慮防守。
   
   (2) 盡量維持門清(不做吃碰槓)，除非吃碰槓能帶來的效益很大
   
   (3) 權重公式: `(8 - 向聽數)x1000 + 役種距離加成(約100~500之間) + 寶牌加成(100) + 有效張數x10 + max(防守程度x牌的安全程度x威脅程度)`

```
   註1: 防守權重是對於所有其他玩家的值取max

   註2: 防守程度是100 x d，其中d與自己的向聽數成正相關，0 <= d <= 1

   註3: 安全程度是15 x s，其中s可以利用之前的規則(不能胡自己打過的牌)來判斷，0 <= s <= 100

   註4: 威脅程度是t，根據別人可能聽牌的程度而定(和副露數成正相關，確定聽牌就是1)，0 <= t <= 1
```
   
4. 高分AI

   (1) 計算一些常見高分役種組合，傾向去做這些牌。
   
   (2) 犧牲掉一些速度或向聽數來追求大分。

   (3) 大幅提高寶牌加成
   
   (4) 權重公式: `(8−向聽數)×180 + 打點潛力分數x信心值  + 寶牌加成(1200)`
   
   `註:  打點潛力分數和與高分役種的湊法有關、信心值和與高分役種的距離有關`

最終實驗方法 - 

1. 基礎邏輯驗證 : 在對戰中放入三隻隨機AI和一隻向聽數AI，互相對戰2500局，驗證穩定往胡牌走優於完全隨機。 紀錄的數據為:胡牌率。
2. 進階邏輯驗證 : 在對戰中放入三隻向聽數AI和一隻進階邏輯(共有三種，分別做)，互相對戰2500局，做三次取數據的平均，驗證進階邏輯數據優於基礎邏輯，且三隻進階邏輯的特色數據互相比較後會較符合自己的特徵。 紀錄的數據為:胡牌率、放槍率、平均單局打點、平均胡牌輪數。

   ```
   註:

   預期結果 :
   
   速度AI的平均胡牌輪數應為最低，且至少較其他進階邏輯皆低於一至二輪以上。
   
   防守AI的放槍(打的牌讓別人胡)應為最低，且至少較其他進階邏輯低於3%以上。
   
   高分AI的平均打點應為最高，且至少較其他進階邏輯高500分以上
   
   ```

3. 策略比較: 在對戰中全部放入三隻不同進階邏輯AI和一隻向聽數AI，互相對戰2000場，利用麻將遊戲的段位分數公式評價策略的優劣性。 紀錄數據為:平均單場分數、胡牌率、放槍率、平均胡牌輪數、平均順位(排名)、根據遊戲段位公式算出來的平均分數(表示平均每場能在遊戲中段位加多少分)。

   (2) 防守型的平均胡牌輪數反而變成最低的，推測跟上個實驗原因類似，在防守型較快聽牌的時候行為接近速度型，但在別人聽牌後很可能棄胡，對戰到速度更快的速度型這種情況可能更明顯，導致防守型只有在真的對局很前面就聽牌的情況下才較可能胡牌，這也體現在防守型胡牌率偏低的數據上。

   (3) 胡大牌型看起來過度的去追求高分的牌型，導致放槍率最高，且平均胡牌輪數也是最高，看起來在日本麻將中並非太好的策略。
 


### 使用方式
<!-- 如何編譯、執行、使用你的程式 -->

本專案是一個基於 C++ 開發的日本麻將 AI 自動化對局模擬與策略評估系統，並結合 Python (Pandas & Matplotlib) 進行實驗數據的統計與動態圖表生成。以下說明如何建立編譯環境、執行模擬。

1. 環境需求

C++ 編譯環境：支援 C++17 或以上標準的編譯器（如 g++ 於 WSL/Linux 環境）。
Python 環境：Python 3.8+，並需安裝數據分析套件。安裝指令如下：

`pip install pandas matplotlib`

2. 編譯 C++ 模擬器

本專案支援使用 Makefile 進行快速編譯。於專案src目錄下直接輸入 make 指令即可自動完成編譯。

指令格式:

`make`

3. 執行AI模擬對局

編譯完成後，可透過命令列參數自訂模擬模式、玩家陣容以及對局數量。

指令格式:

`./mahjong_game [mode] [AI_0] [AI_1] [AI_2] [AI_3] [total_count] [output_csv_path]`

參數說明:

mode:可選擇 round (單局）或 match（整場）。

AI_0 到 AI_3:指定四位玩家的 AI 策略類型。可選參數包含 shanten（向聽型）、speed（速度型）、defense（防守型）、highscore（胡大牌型）、random（隨機亂打型）。

total_count:模擬的總局數或總場數。

output_csv_path:實驗數據輸出的 CSV 檔案路徑。

使用範例 A（單局模式基準測試，1 個向聽 AI 對抗 3 個隨機 AI，跑 1000 局):

`./mahjong_game round shanten random random random 1000 out/base_verify.csv`

使用範例 B（完整整場對局測試，四個AI對抗，跑 500 場完整比賽):

`./mahjong_game match defense speed highscore shanten 500 out/match_results.csv`

4. 數據視覺化繪圖 

本專案提供 Python 腳本，能夠讀取多個實驗生成的 CSV 檔案進行數據合併與平均計算，並支援玩家群組融合功能，分在同一組的將會把數據取平均後再繪圖。

指令格式:

`python src/plot_average_results.py [csv1] [csv2 ...] [--groups num_groups g1 g2 ...] [output_base_name]`

使用範例 A（一般繪圖，直接平均所有對局並繪製各項指標圖表):

`python3 src/plot_average_results.py out/run1.csv out/run2.csv out/run3.csv charts/avg_out`

使用範例 B（分組繪圖，將 Player 2 與 Player 3 的隨機玩家融合成 Group 平均，專注分析 Player 0 與 1):

`python3 src/plot_average_results.py out/run1.csv out/run2.csv out/run3.csv --groups 3 0 1 2,3 charts/grouped_out`
